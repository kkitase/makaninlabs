# Makanin Labs サイト改修設計：AI × Marketing トレーニング送客への転換

- **日付**: 2026-05-02
- **対象**: makaninlabs.com（Firebase Hosting、`block-drop-le7hp`）
- **スコープ**: トップページのリブランドと日本語化（Phase 1）

---

## 1. 背景と目的

現状のサイトは「AI Marketing Solutions」を打ち出すコーポレート LP だが、サービスの輪郭が曖昧で、何を依頼できるのかが伝わりにくい。

オーナーは AI × Marketing 領域での講演・登壇・カスタムワークショップ実績を持っており、これからは **「講演・ワークショップの依頼を受けるサイト」** として明確に位置づけ直したい。

更新工数を最小化するため、初期フェーズは **トップページの全面改修のみ** に絞る。Insights / Slides 等の蓄積コンテンツは将来フェーズに分離する。

## 2. 方針

### 採用する構成
- 既存の単一ページ構成を維持（React Router 不要）
- 日本語のみ（JP/EN 切替を全削除）
- ブランドブランド（Makanin Labs）主体。講師個人を前面に出さない
- 4 セクション構成：Hero / Topics / Workshop / Contact

### 専門領域（看板）
1. AI 導入の組織変革
2. AI を活用したコンテンツ制作
3. ツール・プロンプト実践（ChatGPT / Claude / Gemini 等）

### 技術スタック（変更なし）
- React 19 / TypeScript / Vite 6 / Tailwind CSS（CDN）/ Firebase Hosting
- 新規 npm パッケージ追加なし
- ブランドカラー（`#333333` / `#F9F9F9` / `#FFE600`）と書体（serif）は維持

## 3. ページ構造

### 縦積み順序（トップページ）
1. **Header** — ロゴ + CTA ボタン（"ご相談"）のみ。ナビリンクなし
2. **Hero** — メインメッセージ + CTA（"講演・ワークショップを依頼する"）
3. **Topics** — 3 つの専門領域（カード 3 列）
4. **Workshop** — 2 つの依頼形態（カード 2 列）
5. **Contact** — 既存の Google フォーム
6. **Footer** — ロゴ + tagline + コピーライト

### スコープ外（明示）
- `/insights`、`/slides` ページの作成
- 講師紹介セクション
- 数値による Track Record 表示
- React Router 導入、Markdown ローダー、ヘッドレス CMS
- ブランドカラーやタイポグラフィの変更
- Firebase Hosting 設定変更

## 4. セクション別設計

### 4.1 Header
- 削除：JP/EN 言語切替トグル全体
- 削除：`lang` / `setLang` 関連の props
- 維持：ロゴ（"M" 黒四角 + "Makanin Labs"）
- 維持：右上 CTA ボタン "ご相談" → `#contact` アンカー
- ナビリンク：**追加しない**（ミニマル維持）

### 4.2 Hero

**メインコピー（採用案：B「AI を理解した人を、増やす」）**

> AI 時代のマーケターを、現場で育てる。
>
> 生成 AI は触れる人がいて初めて武器になります。Makanin Labs は、企業のマーケティング組織に向けた講演・カスタムワークショップで、"明日から使える" 知識と判断軸を提供します。

- バッジ表記：`私たちが目指すこと`
- CTA ボタン：**"講演・ワークショップを依頼する"** → `#contact`
- 視覚装飾（イエロー × グレーの blur グラデ、`Scroll` インジケーター等）は維持
- **タイトルの組版マッピング**（既存の 3 行構造 `title` / `titleSecond` / `titleAccent` を流用）：
  - 1 行目（`title`）：`AI 時代のマーケターを、`
  - 2 行目（`titleSecond`）：`現場で`
  - 3 行目（`titleAccent`、イタリック + イエロー左罫線）：`育てる。`
- 説明文：上記コピーの 2 段落目をそのまま使用

### 4.3 Topics

- **見出し**：`Topics`
- **サブヘッド**：「私たちが講演・ワークショップで扱う 3 つの領域」
- **レイアウト**：3 カラム（既存の Services セクションを流用）

#### カード 1: AI 導入の組織変革
> **組織と業務に AI を組み込む**
>
> ツール導入だけでは現場は動きません。AI を "誰が・何に・どう使うか" を業務フローに織り込み、属人化させずに組織として活用するための仕組みづくり。マーケ組織の変革プロセスを設計から伴走まで支援します。

#### カード 2: AI を活用したコンテンツ制作
> **品質を落とさず、制作を加速する**
>
> コピー・ビジュアル・動画・SNS 投稿。生成 AI を制作実務に正しく組み込めば、量と速度を両立しながらクリエイティブの質を落とさずに済みます。実務で使える型と判断基準を、ワークショップ形式で提供します。

#### カード 3: ツール・プロンプト実践
> **ChatGPT / Claude / Gemini を使いこなす**
>
> 主要な生成 AI ツールにはそれぞれ得意・不得意があります。マーケ業務のシーン別にどのツールを選び、どんなプロンプトを書くか。ハンズオン形式で実務に直結するスキルを身につけます。

#### アイコン
- カード 1：ネットワーク / 組織図系（Heroicons outline `users` 系または `building-office`）
- カード 2：編集ペン / 書類系（既存の鉛筆アイコン流用可）
- カード 3：コマンド / ターミナル / 歯車系（Heroicons outline `command-line` 系）

### 4.4 Workshop（新規セクション）

- **見出し**：`Workshop`
- **サブヘッド**：「貴社の課題に合わせた、カスタム形式で提供します」
- **レイアウト**：2 カラム

#### カード 1: 講演・登壇（Keynote / Speaking）
> **60 〜 90 分の単発講演・登壇**
>
> 社内向けの経営層・全社イベントから、業界カンファレンス・パートナー向けセッション・ウェビナーまで対応。AI とマーケティングの現在地、組織がいま向き合うべき問いを、実践的な事例と共に語ります。
>
> 想定: 経営会議 / 全社キックオフ / 業界カンファレンス / パートナー向けセッション

#### カード 2: カスタムワークショップ（Workshop）
> **2 時間 〜 1 日 / 貴社の課題に合わせて設計**
>
> 決まったコースは提供していません。貴社のマーケ組織の状況・課題・対象者をヒアリングした上で、内容を 1 から設計するハンズオン型ワークショップ。3 つの専門領域（組織導入 / コンテンツ制作 / ツール実践）から組み合わせます。
>
> 想定: マーケ部門 5 〜 30 名 / 短時間セミナー〜半日・1 日研修 / オンライン or 対面

#### 補助テキスト（セクション最下部）
> ご依頼内容・規模・予算など、まずはお気軽にご相談ください。
>
> → Contact セクションへの導線

### 4.5 Contact

### 4.6 Footer

- 維持：「Towards Sincerity」「crafted with sincerity.」の装飾コピー
- 削除：英語版コンテンツと `Language` 切替対応
- 更新：コピーライト年を `© 2026` に

### 4.7 `index.html` メタ情報
- `<title>`：`Makanin Labs - AI × Marketing 講演・ワークショップ`

- `<html lang>`：`en` → `ja`
- OGP（`og:title` / `og:description` / `og:image`）：title・description を上記に揃える。OGP 画像は現状のものを維持

## 6. データモデル

すべてのテキストは TypeScript リテラルとしてコンポーネント内に直接記述する（バイリンガルロジック削除のため、`content[lang]` 構造は不要）。

```ts
// 例：Topics.tsx
const items = [
  {
    title: 'AI 導入の組織変革',
    subtitle: '組織と業務に AI を組み込む',
    description: '...',
  },
  // ...
];
```

外部データソース（CMS、Markdown、JSON）は使用しない。

## 7. エラーハンドリング・バリデーション

- フォームは既存の `required` 属性 + `type="email"` のブラウザバリデーションを維持
- Google フォーム送信は `iframe target` への POST 形式で、ネットワークエラー時は表示上は「送信完了」になる仕様（既存挙動）— 今回スコープでは変更しない
- 別セッションで取り扱うべき改善事項として **将来課題** に記録

## 8. テスト方針

- 単体テスト：導入しない（既存もテストフレームワークなし）
- 動作確認：
  - `npm run dev` で `http://localhost:3000` を起動し、目視で全セクションが表示されることを確認
  - レスポンシブ：モバイル（375px）/ タブレット（768px）/ デスクトップ（1280px）で崩れがないか確認
  - フォーム送信：実際に Google フォームへテスト送信し、レスポンスシートに記録されるか確認
  - リンク：CTA ボタン群が `#contact` に正しくスクロールするか確認
- ビルド検証：`npm run build` がエラーなく完了し、`dist/` が生成されることを確認

## 9. デプロイ

- 既存の Firebase Hosting にデプロイ：
  ```bash
  npm run build
  firebase deploy --only hosting
  ```
- ステージング環境はないため、デプロイ前に必ずローカルで `npm run preview` による最終確認を行う
